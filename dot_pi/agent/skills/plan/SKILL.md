# Plan

Before executing, produce a concise plan covering:

1. **Goal** — one-line summary of what we're building/fixing
2. **Files** — list of files to create, modify, or read
3. **Steps** — numbered steps in dependency order
4. **Risks** — anything that could break or go wrong

Show the plan to the user for approval, then execute step by step.
